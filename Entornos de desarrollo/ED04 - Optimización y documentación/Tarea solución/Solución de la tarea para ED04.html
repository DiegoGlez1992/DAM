REFACTORIZACIÓN

Descarga el proyecto "Deposito" pulsando aquí.

Abre  el proyecto.

Para la resolver la primera actividad, vamos a proceder a refactorizar el proyecto  usando las herramientas que incorpora NetBeans:

1. Las clases deberán formar parte del paquete cuentas.

Entras en NetBeans y cuando te pida el nombre del proyecto y ubicación:

Desactiva "create Main Class" para que no te cree ningún paquete ni ninguna clase Principal:

Pulsas en "Finish". Una vez que estás ahí, pulsas sobre el botón derecho del ratón, donde pone "Source Packages" y eliges: new/Java package y pones el nombre de "cuentas":

En dicho paquete pegas los dos ficheros .java que hay en el proyecto "Deposito" que te has bajado anteriormente.

Te da un error de codificación porque tienes que poner al principio: "package cuentas";

También puedes poner los dos ficheros .java en "Source packages" y luego pulsas el botón derecho del ratón sobre cada fichero con extensión .java y seleccionar /Refactorizar/Mover. Finalmente, indicar que se quiere mover al paquete cuentas. Y el IDE te pone automáticamente: "package cuentas";

2. Cambiar Nombre de la variable "miCuenta" por "cuenta1".

Dicha variable está en la clase "Main". 

Seleccionar la variable miCuenta y pulsar el botón derecho del ratón, a continuación en el menú contextual pinchar la opción /Refactor/Rename:

Te visualizará esta ventana:

Y le pones el nombre de "cuenta1".

3. Introducir el método operativa_cuenta, que englobe las sentencias de la clase Main que operan con el objeto cuenta1.

Seleccionar todo el código del método main y pulsar botón derecho del ratón, en el menú contextual pulsar la opción /Refactor/Introduce/metodo:
Y le ponemos el nombre: "operativa_cuenta" y  elegimos acceso público

Como se aprecia en la siguiente imagen, el código seleccionado, ha sido sustituido por el nuevo método.

4. Encapsula los atributos de la clase CCuenta.

Hay dos alternativas.
Encapsulado atributo a atributo: seleccionar el atributos en el código y marcar la opción de menú Refactor/Encapsulate Fields en el menú principal y pinchamos en las casillas correspondientes para crear el set y get. Habrá que repetir el proceso cuatro veces.

Encapsulado múltiple: En la ventana anterior puede seleccionar "Select all" y generaría los set y get de todos los atributos.

Los cuatro campos seleccionados, pasa a disponer de métodos getter y setter para acceder a sus valores.

Como se aprecia en la siguiente imagen, los campos disponen de los métodos para leer y escribir su valor.

GIT

1. Configurar GIT para el proyecto. Crear un repositorio público en GitHub

La solución propuesta utiliza un repositorio en la máquina Linux.

Si todavía no está instalado GIT desde un terminal de Linux con permiso de administrador:

    • apt-get update
    • apt-get install git

Crear un directorio donde trabajar con el repositorio. P.e. /home/debian/Escritorio/deposito.

Asignar permisos de acceso al directorio, como administrador Linux, comando chmod 777 /home/debian/Escritorio/deposito

Nota: en un entorno real, en lugar de dar permisos de acceso completos, habría que aplicar sólo aquellos estrictamente necesarios.

En Eclipse, pulsar el botón derecho del ratrón sobre el proyecto y seleccionar Team/Compartir proyecto/ 

2. Realizar, al menos, una operación commit, comentando el resultado de la ejecución.

Volcar la información del área de trabajo al repositorio local-Eclipse con la opción Team/Commit.

Previo al commit, habrá que seleccionar los ficheros del proyecto candidatos a formar parte del repositorio (Unstaged  =>  Staged): seleccionarlos + Botón derecho + Add to Index.

Finalmente, introducir un mensaje y pulsar el botón Commit.

Cada vez que se realizan modificaciones del código fuente, se generan nuevas versiones del fichero que estemos modificando, si realizamos una operación commit, también vamos a actualizar el repositorio (depósito).

3. Mostrar el historial del versiones para el proyecto mediante un comando desde consola, comentar el resultado.

Desde un terminal Linux, posicionarnos en la ruta del repositorio /home/debian/Escritorio/deposito, ejecutar el comando git log.

1. Inserta comentarios Javadoc en la clase CCuenta.

A nivel de clase incluir al menos las etiquetas:

    • @autor
    • @version
    • @since

En los métodos considerar las etiquetas:

    • @param
    • @return
    • @throws

A continuación se muestra parte de la clase CCuenta, con comentarios Javadoc:

package cuentas;

public class CCuenta {


private String nombre;
private String cuenta;
private double saldo;
private double tipoInterés;

public CCuenta()
{
}

public CCuenta(String nom, String cue, double sal, double tipo)
{
nombre =nom;
cuenta=cue;
saldo=sal;
}

public double estado()
{
return saldo;
}
/**
* 
* @param cantidad Es el dinero a ingresar en la cuenta
* @throws Exception Lanza este error si la cantida recibida es negativa
*/
public void ingresar(double cantidad) throws Exception
{
if (cantidad<0)
throw new Exception("No se puede ingresar una cantidad negativa");
saldo = saldo + cantidad;
}
/**
* 
* @param cantidad Es el dinero a quitar de la cuenta
* @throws Exception Lanza este error si la cantidad a retirar es negativa o es superios al dinero que hay en la cuenta
*/
public void retirar(double cantidad) throws Exception
{
if (cantidad <= 0)
throw new Exception ("No se puede retirar una cantidad negativa");
if (estado()< cantidad)
throw new Exception ("No se hay suficiente saldo");
saldo = saldo - cantidad;
}

/**
* @return the cuenta
*/
public String getCuenta() {
return cuenta;
}

/**
* @return the nombre
*/
public String getNombre() {
return nombre;
}

/**
* @return the saldo
*/
public double getSaldo() {
return saldo;
}

/**
* @return the tipoInterés
*/
public double getTipoInterés() {
return tipoInterés;
}

/**
* @param cuenta the cuenta to set
*/
public void setCuenta(String cuenta) {
this.cuenta = cuenta;
}

/**
* @param nombre the nombre to set
*/
public void setNombre(String nombre) {
this.nombre = nombre;
}

/**
* @param saldo the saldo to set
*/
public void setSaldo(double saldo) {
this.saldo = saldo;
}

/**
* @param tipoInterés the tipoInterés to set
*/
public void setTipoInterés(double tipoInterés) {
this.tipoInterés = tipoInterés;
}
}
